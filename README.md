<h2 id="features">Features</h2>

<h3>Frontend</h3>
<ul>
  <li>Built with <strong>Nuxt 3</strong> (Vue.js framework)</li>
  <li>Styled with <strong>CSS 3</strong>, <strong>GSAP</strong> animations, and <strong>Font Awesome</strong> icons</li>
  <li>Stripe payment integration for a dynamic "Pay Me" page</li>
  <li>Responsive design for mobile and desktop views</li>
  <li>Multi-language support with a custom language switcher</li>
</ul>

<h3>Backend</h3>
<ul>
  <li><strong>Express.js</strong> server handling API requests</li>
  <li><strong>Stripe API</strong> for payment handling</li>
</ul>

<h3>Deployment</h3>
<ul>
  <li><strong>Frontend:</strong> Static pages generated by <strong>Nuxt 3</strong> in the <code>/app</code> folder, deployed on <strong>Infomaniak</strong></li>
 <li><strong>Backend:</strong> API also hosted on <strong>Infomaniak</strong> (previously on O2Switch). It's possible to deploy the backend on a separate server using a dedicated folder. On Infomaniak, deployment is done via <strong>SSH</strong></li>
</ul>

<h2 id="installation">Installation</h2>

<p>To get started with this project, follow the steps below:</p>

<h3>1. Clone the repository</h3>
<p>First, clone the repository to your local machine:</p>
<pre><code>git clone https://github.com/Djason16/djason-cv.git</code></pre>

<h3>2. Navigate to the project directory</h3>
<p>Go to the newly cloned project directory:</p>
<pre><code>cd djason-cv</code></pre>

<h3>3. Set up environment variables</h3>

<h4>Backend</h4>
<p>In the <code>/api</code> folder, create two environment files for different environments:</p>

<p><strong>.env</strong> for development:</p>
<pre><code>
NODE_ENV=development
BACKEND_DOMAIN=http://localhost:3001
FRONTEND_DOMAIN=http://localhost:3000
STRIPE_PUBLIC_KEY=your-stripe-public-key
STRIPE_SECRET_KEY=your-stripe-secret-key
</code></pre>

<p><strong>.env.production</strong> for production:</p>
<pre><code>
NODE_ENV=production
BACKEND_DOMAIN=https://your-website.com
FRONTEND_DOMAIN=https://your-website.com
STRIPE_PUBLIC_KEY=your-stripe-public-key
STRIPE_SECRET_KEY=your-stripe-secret-key
</code></pre>

<h4>Frontend</h4>
<p>In the <code>/app</code> folder, create a folder named <code>env</code>. Inside this <code>/env</code> folder, create two JavaScript files:</p>

<p><strong>env.dev.js</strong> for development:</p>
<pre><code>
export default {
  BACKEND_DOMAIN: "http://localhost:3001",
  FRONTEND_DOMAIN: "http://localhost:3000",
  STRIPE_PUBLIC_KEY: "your-stripe-public-key",
  STRIPE_SECRET_KEY: "your-stripe-secret-key"
};
</code></pre>

<p><strong>env.prod.js</strong> for production:</p>
<pre><code>
export default {
  BACKEND_DOMAIN: "https://your-website.com",
  FRONTEND_DOMAIN: "https://your-website.com",
  STRIPE_PUBLIC_KEY: "your-stripe-public-key",
  STRIPE_SECRET_KEY: "your-stripe-secret-key"
};
</code></pre>

<h3>4. Install dependencies</h3>

<h4>Backend</h4>
<p>Navigate to the <code>/api</code> folder:</p>
<pre><code>cd api</code></pre>
<p>Install the dependencies:</p>
<pre><code>npm install</code></pre>

<h4>Frontend</h4>
<p>Navigate to the <code>/app</code> folder:</p>
<pre><code>cd app</code></pre>
<p>Install the dependencies:</p>
<pre><code>npm install</code></pre>

<h3>5. Run the Development Server</h3>

<h4>Backend</h4>
<p>After installing the dependencies, run the backend server:</p>
<pre><code>npm run dev</code></pre>
<p>This will start the Express.js server, usually accessible at <a href="http://localhost:3001">http://localhost:3001</a> in development mode.</p>

<h4>Frontend</h4>
<p>Run the Nuxt.js development server:</p>
<pre><code>npm run dev</code></pre>
<p>The frontend should now be running, typically at <a href="http://localhost:3000">http://localhost:3000</a> in development mode.</p>

<h4>Alternative: run both frontend and backend with one command</h4>

<p>If you prefer to start both the frontend and backend at once, you can use the included script:</p>

<pre><code>start.bat</code></pre>

<p>Just double-click <code>start.bat</code> from the root folder (where <code>/app</code> and <code>/api</code> are located).</p>

<p>This will open two separate terminal windows:</p>
<ul>
  <li>One for <code>app</code> running <code>npm run dev</code> (Nuxt frontend)</li>
  <li>One for <code>api</code> running <code>npm run dev</code> (Express backend)</li>
</ul>
<p>You can stop the servers by closing their respective terminal windows.</p>

<h3>6. Build for Production (Optional)</h3>
<p>Once you're ready to deploy or want to test the production version locally, follow these steps for the frontend (<code>/app</code> folder):</p>

<h4>Option 1: Build (for dynamic applications)</h4>
<p>If you prefer a dynamic production build (with server-side rendering), run:</p>
<pre><code>npm run build</code></pre>
<p>This will build the frontend for production, optimizing the files for deployment. After building, you can start the Nuxt server:</p>
<pre><code>npm run start</code></pre>
<p>This will serve your app with server-side rendering (SSR), and your site will be live on your server.</p>

<h4>Option 2: Generate (for static sites)</h4>
<p>If you prefer a static site, which is great for performance and can be easily deployed to static hosting providers, run:</p>
<pre><code>npm run generate</code></pre>
<p>This will create a static version of your site in the <code>dist/</code> folder. You can deploy the contents of the <code>dist/</code> folder to a web server like <strong>Infomaniak</strong> or any other hosting provider that supports static sites.</p>

<h3>7. Deploy the Frontend (Optional)</h3>
<p>Once you've built or generated your frontend (via the above steps), it’s time to deploy it:</p>

<h4>For Dynamic Sites (SSR)</h4>
<p>If you used <code>npm run build</code> and <code>npm run start</code> in Step 6, you can deploy the dynamic site to a server that supports Node.js, and it will serve the pages with SSR.</p>

<h4>For Static Sites</h4>
<p>If you used <code>npm run generate</code> in Step 6, upload the contents of the <code>dist/</code> folder to a static hosting service such as <strong>Infomaniak</strong>.</p>

<h3>8. Deploy the Backend (Optional)</h3>

<h4>Prepare your backend</h4>
<p>Ensure that your backend is working as expected locally, with all environment variables set up correctly in the <code>/api</code> folder (e.g., in <code>.env</code> or <code>.env.production</code>).</p>

<h4>Deploy the backend to a hosting provider</h4>
<p>For <strong>O2Switch</strong> or similar providers, you'll need to upload your <code>/api</code> folder (the Node.js backend) to your server.</p>
<p>Make sure the server is set up to run the backend properly. You might need to configure your hosting provider to handle <strong>Node.js</strong> applications.</p>
<p>If you're using a <strong>VPS</strong> or a similar solution, ensure that your server is running the backend using <strong>Node.js</strong>, and you’ve configured the correct environment variables for production.</p>

<h4>Start the backend</h4>
<p>Once your backend is deployed, you can start the server with:</p>
<pre><code>node server.js</code></pre>
<p>Or, depending on your hosting provider, follow their instructions to start the backend service.</p>

<h4>Configure your frontend to connect to the deployed backend</h4>
<p>After deploying the backend, ensure that the frontend is correctly configured to point to your live backend. To do this:</p>
<ul>
  <li>Check that the <strong>frontend environment variables</strong> in the <code>/app/env/env.prod.js</code> file are properly updated to reflect the live backend URL:</li>
</ul>
<pre><code>
export default {
  BACKEND_DOMAIN: "https://your-website.com",
  FRONTEND_DOMAIN: "https://your-website.com",
  STRIPE_PUBLIC_KEY: "your-stripe-public-key",
  STRIPE_SECRET_KEY: "your-stripe-secret-key"
};
</code></pre>
<p>Similarly, verify that the <strong>backend environment variables</strong> in <code>/api/.env.production</code> match the production URL of your backend:</p>
<pre><code>
NODE_ENV=production
BACKEND_DOMAIN=https://your-website.com
FRONTEND_DOMAIN=https://your-website.com
STRIPE_PUBLIC_KEY=your-stripe-public-key
STRIPE_SECRET_KEY=your-stripe-secret-key
</code></pre>
<p>This ensures that both the frontend and backend are connected and using the correct environment variables.</p>
